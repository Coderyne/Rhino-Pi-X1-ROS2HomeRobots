# HA 语音助手 — 部署指南

将语音助手部署到**X1派**的完整步骤。

---

## 一、环境要求

| 项目 | 要求 |
|------|------|
| 架构 | ARM aarch64 |
| 平台 | Qualcomm QCS8550 |
| 外设 | USB 麦克风 + 音箱 |
| Python | >= 3.8 |

---

## 二、准备工作

### 2.1 确认 AidLux 授权状态

```bash
aid-lms-check
```

输出应包含 `"isActive": true`。若未激活，参考 [AidLux 授权激活文档](https://rhinopi.docs.aidlux.com/software/aidlux/developer-guide/developer-guide-activation) 激活。

### 2.2 安装 AidVoice SDK

```bash
sudo aid-pkg update
sudo aid-pkg install aidlite-sdk
sudo aid-pkg install aidlite-qnn236
sudo aid-pkg -i aidvoice-sdk
```

验证安装：

```bash
ls -l /usr/local/lib/libaidvoice_speech.so
ls /usr/local/include/aidlux/aidvoice/aidvoice_speech.hpp
```

### 2.3 安装 Python 依赖

```bash
pip3 install sherpa-onnx sounddevice requests numpy wyoming
```

### 2.4 安装系统音频库

```bash
sudo apt update
sudo apt install -y alsa-utils libportaudio2 pulseaudio
```

## 三、编译 asr_bridge.so

`asr_bridge.so` 是连接 Python 和 AidVoice C++ SDK 的桥梁，需要从源码编译。

### 3.1 准备源码

两个源码文件已包含在项目目录中：

```bash
ls -l ~/HA_VoiceAssistant/asr_bridge.cpp
ls -l ~/HA_VoiceAssistant/asr_bridge.h
```

### 3.2 编译

```bash
cd ~/HA_VoiceAssistant

g++ -std=c++17 -fPIC -shared \
    -I/usr/local/include \
    -L/usr/local/lib \
    -laidvoice_speech \
    -lpthread \
    -o asr_bridge.so \
    asr_bridge.cpp
```

### 3.3 验证编译结果

```bash
file asr_bridge.so
# 预期输出: ... ELF 64-bit LSB shared object, ARM aarch64 ...

# 检查符号
nm -D asr_bridge.so | grep asr_bridge_create
# 应输出类似: 0000000000000xxx T asr_bridge_create
```

## 五、配置 Home Assistant

### 5.1 创建 HA 长期访问令牌

1. 浏览器打开 `http://目标机器IP:8123`
2. 进入 **设置 → 长期访问令牌 → 创建令牌**
3. 复制令牌内容

### 5.2 写入 keys.txt

```bash
echo "HA_TOKEN=你创建的令牌" > ~/HA_VoiceAssistant/keys.txt
```

### 5.3 确认对话代理

启动脚本默认使用 agent_id：`conversation.qwen3_4b_8550`。如果 HA 中配置了其他对话代理，启动时需指定：

```bash
./start.sh --ha-agent-id conversation.你的代理id
```

## 六、音频设备配置

### 6.1 检测录音设备

```bash

```

```
card 1: kalamaqrdsndcar [kalama-qrd-snd-card], device 5: CODEC_DMA-...
```

麦克风通常是 USB 设备（card 0），记下 `card X, device Y`。

### 6.2 检测播放设备

```bash

```

音箱通常是 USB 或板载音频输出，记下 `card X, device Y`。

### 6.3 修改启动脚本

编辑 `start.sh`，修改设备名：

```bash

DEVICE_NAME="plughw:0,0"

# 播放设备（更换为检测到的设备）

```

## 八、配置唤醒词

唤醒词使用 sherpa-onnx 的 Keyword Spotting 引擎，模型已预训练好，只需提供关键词列表即可。预设唤醒词为 "小爱同学"。

### 8.1 关键词文件位置

```
~/HA_VoiceAssistant/models/sherpa-onnx-kws-zipformer-wenetspeech-3.3M-2024-01-01/
├── keywords.txt        # 模型使用的唤醒词文件
├── keywords_raw.txt    # （可选）原始唤醒词，便于修改
└── tokens.txt          # 拼音 token 映射表
```

### 8.2 文件格式说明

`sherpa-onnx-kws-zipformer-wenetspeech-3.3M-2024-01-01` 模型使用**拼音 token** 表示唤醒词。

**`keywords.txt`**（模型直接使用）：

```
x iǎo ài t óng x ué @小爱同学
```

每行格式：`拼音token（空格分隔） @ 显示文本`

**`keywords_raw.txt`**（供 `text2token` 工具使用）：

```
小爱同学 @小爱同学
```

> 关键词中不能包含空格，多词组合请用下划线 `_` 连接。

### 8.3 使用 text2token 工具生成

`sherpa_onnx` pip 包自带 `sherpa-onnx-cli text2token` 工具，可将中文文本自动转为拼音 token：

**步骤 1**：创建 `keywords_raw.txt`

```bash
cd ~/HA_VoiceAssistant/models/sherpa-onnx-kws-zipformer-wenetspeech-3.3M-2024-01-01

cat > keywords_raw.txt << 'EOF'
小爱同学 @小爱同学
EOF
```

**步骤 2**：转换为 `keywords.txt`

```bash
sherpa-onnx-cli text2token \
  --tokens tokens.txt \
  --tokens-type ppinyin \
  keywords_raw.txt keywords.txt
```

**步骤 3**：验证生成结果

```bash
cat keywords.txt
# 输出：
# x iǎo ài t óng x ué @小爱同学
```

### 8.4 替换并重启

```bash
# 确认 keywords.txt 已更新
cp keywords.txt ~/HA_VoiceAssistant/models/sherpa-onnx-kws-zipformer-wenetspeech-3.3M-2024-01-01/

# 重启语音助手
~/HA_VoiceAssistant/./start.sh
```

### 8.5 调整唤醒灵敏度

通过 `--kws-threshold` 参数控制：

```bash
# 阈值越低越灵敏（默认 0.10）
./start.sh --kws-threshold 0.08    # 更灵敏
./start.sh --kws-threshold 0.15    # 更迟钝（减少误唤醒）
```

> 如果自定义唤醒词后始终无法触发，可尝试降低阈值。

### 8.6 多唤醒词建议

- 每个唤醒词建议 2~4 个汉字，太短容易误唤醒，太长用户说着累
- 中文唤醒词直接写汉字即可，工具会自动转为拼音 token
- 英文唤醒词需使用 BPE 模型（本项目模型仅支持中文）

## 九、常见问题

### Q: TTS 播放无声音

```
Expression 'alsa_snd_pcm_prepare( stream->playback.pcm )' failed
```

**原因**：播放设备配置错误或 PulseAudio 未运行。

**解决**：
```bash
# 确认 PulseAudio 运行中
pulseaudio --check

# 如果未运行
pulseaudio --start

# 检查播放设备名称是否正确
python3 -c "import sounddevice as sd; print(sd.query_devices())"
```

### Q: 唤醒不灵敏或误唤醒

**原因**：阈值过低或过高。

**解决**：
```bash
# 调低阈值（更灵敏）
./start.sh --kws-threshold 0.08

# 调高阈值（减少误唤醒）
./start.sh --kws-threshold 0.15
```

### Q: VAD 不触发，ASR 不输出

**原因**：VAD 能量阈值与麦克风不匹配。

**解决**：编辑 `asr_aidvoice.py`，调整 `_SPEECH_ENERGY_THRESHOLD` 值：

```python
_SPEECH_ENERGY_THRESHOLD = 0.02  # 默认值，麦克风噪音大可调高到 0.05
```
