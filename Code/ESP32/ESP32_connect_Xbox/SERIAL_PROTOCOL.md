# Xbox 手柄串口通信协议

## 概述

ESP32-C3 通过蓝牙 BLE 连接 Xbox Series X 手柄后，将手柄的摇杆、扳机、按键数据封装为 11 字节的二进制帧，通过 Serial1（GPIO 20 TX / GPIO 21 RX，115200 8N1）每约 10ms 发送一帧。

## 数据帧格式

| 字节偏移 | 字段 | 类型 | 说明 |
|---------:|------|------|------|
| 0 | header[0] | uint8 | 包头，固定值 `0x07` |
| 1 | header[1] | uint8 | 包头，固定值 `0x21` |
| 2 | joyL_X | uint8 | 左摇杆 X，原始值 -100~100，传输值 28~228（+128 偏移） |
| 3 | joyL_Y | uint8 | 左摇杆 Y，原始值 -100~100，传输值 28~228（+128 偏移） |
| 4 | joyR_X | uint8 | 右摇杆 X，原始值 -100~100，传输值 28~228（+128 偏移） |
| 5 | joyR_Y | uint8 | 右摇杆 Y，原始值 -100~100，传输值 28~228（+128 偏移） |
| 6 | trig_L | uint8 | 左扳机 LT，范围 0~100 |
| 7 | trig_R | uint8 | 右扳机 RT，范围 0~100 |
| 8 | buttons_1 | uint8 | 按钮组 1（位域打包） |
| 9 | buttons_2 | uint8 | 按钮组 2（位域打包） |
| 10 | checksum | uint8 | 前 10 字节求和取低 8 位 |

## 按钮位域

### buttons_1（字节 8）

| 位 | 按键 | 说明 |
|--:|------|------|
| 0 | A | 按下为 1 |
| 1 | LB | 左肩键 |
| 2 | RB | 右肩键 |
| 3 | Xbox | Xbox 导航键 |
| 4 | - | 保留 |
| 5 | - | 保留 |
| 6 | - | 保留 |
| 7 | - | 保留 |

### buttons_2（字节 9）

| 位 | 按键 | 说明 |
|--:|------|------|
| 0 | B | 按下为 1 |
| 1 | X | 按下为 1 |
| 2 | Y | 按下为 1 |
| 3 | - | 保留 |
| 4 | - | 保留 |
| 5 | - | 保留 |
| 6 | - | 保留 |
| 7 | - | 保留 |

## 摇杆数据换算

手柄原始摇杆值为 `uint16_t`（范围 0~65535），中心约为 32768。发送方将其归一化到 -100~100 后偏移 128 写入帧。

### 发送方编码（ESP32）

```
joy_value = (int)((raw - 32768.0) / 32768.0 * 100.0)  // X 轴
joy_value = -(int)((raw - 32768.0) / 32768.0 * 100.0) // Y 轴（取反）
frame_byte = (uint8_t)(joy_value + 128)
```

- 摇杆 Y 轴经取反处理，因此**向上为正（>128），向下为负（<128）**
- 摇杆 X 轴**向右为正（>128），向左为负（<128）**

### 接收方解码

```
joy_value = (int)frame_byte - 128   // 范围 -100~100，正=右/上，负=左/下
```

## 扳机数据换算

手柄原始扳机值为 `uint16_t`（范围 0~1023）。发送方将其归一化到 0~100 后写入帧。

```
trig_value = (int)(raw / 1023.0 * 100.0)
frame_byte = (uint8_t)(trig_value)  // 直接用原值，无偏移
```

## 校验和计算

```cpp
uint16_t checksum = 0;
for (int i = 0; i < 10; i++) {
    checksum += data[i];
}
data[10] = (uint8_t)(checksum & 0xFF);  // 取低 8 位
```

接收方验证方式：对前 10 个字节求和取低 8 位，与 `data[10]` 比较即可。不考虑进位回卷。

## 完整帧示例

以下为示例帧（假设摇杆居中、扳机松开、无按键按下）：

```
07 21 80 80 80 80 00 00 00 00 48
│     │  │  │  │  │  │  │  │  └─ checksum
│     │  │  │  │  │  │  │  └──── buttons_2 = 0 (B/X/Y 均未按下)
│     │  │  │  │  │  │  └─────── buttons_1 = 0 (A/LB/RB/Xbox 均未按下)
│     │  │  │  │  │  └────────── trig_R = 0
│     │  │  │  │  └───────────── trig_L = 0
│     │  │  │  └──────────────── joyR_Y = 0x80 (128, 居中)
│     │  │  └─────────────────── joyR_X = 0x80 (128, 居中)
│     │  └────────────────────── joyL_Y = 0x80 (128, 居中)
│     └───────────────────────── joyL_X = 0x80 (128, 居中)
└─────────────────────────── header
```

校验和：`0x07 + 0x21 + 0x80×4 + 0 + 0 + 0 + 0 = 0x07 + 0x21 + 0x200 = 0x228`，取低 8 位得 `0x28`。

## 双向通信

协议支持双向数据交换：

- **输出方向（ESP32 → 对端）**：每 10ms 发送一帧上述 11 字节数据
- **输入方向（对端 → ESP32）**：对端发送相同格式的帧（包头 `0x07 0x21`），ESP32 检测到包头后自动触发手柄 250ms 振动，带 300ms 冷却防抖

输入方向的帧只需要正确匹配前 2 字节包头，后续字节不影响振动触发。

## 注意事项

- 帧发送频率约为 100Hz（每 10ms 一帧），接收方需自行处理去抖和死区
- 校验和仅覆盖前 10 字节，不覆盖校验和自身
- 若需使用其他按钮（Select、Start、Share、LS、RS、十字键），可扩展 buttons_2 的保留位
