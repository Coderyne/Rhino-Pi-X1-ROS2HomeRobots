## 方法一：/etc/aidlux/ 脚本目录（推荐，最简单）

系统已配置 `/etc/rc.local` 在开机时自动遍历执行 `/etc/aidlux/` 下的所有可执行脚本。

### 使用步骤

1. 创建脚本文件（例如 `/etc/aidlux/my-startup.sh`）：

```bash
sudo tee /etc/aidlux/my-startup.sh << 'EOF'
#!/bin/bash
nohup your_command_here > /var/log/my-startup.log 2>&1 &
EOF
```

2. 赋予执行权限：

```bash
sudo chmod 775 /etc/aidlux/my-startup.sh
```

3. 重启验证，或手动测试：

```bash
sudo /etc/aidlux/my-startup.sh
```

### 示例：开机自动启动 aidllm API 服务

```bash
# 创建脚本
sudo tee /etc/aidlux/start-aidllm << 'EOF'
#!/bin/bash
nohup aidllm start api -m Qwen3-4B-Instruct-cl4096 > /var/log/aidllm.log 2>&1 &
EOF

# 赋予执行权限
sudo chmod 775 /etc/aidlux/start-aidllm
```

### 注意事项

- 脚本需要以 `#!/bin/bash` 开头，指定解释器
- 后台运行需使用 `nohup ... &` 或 `&`，否则会阻塞开机流程
- 标准输出和错误输出建议重定向到日志文件，方便排查问题
- 如果脚本需要延迟执行，可在命令前加 `sleep N`

## 方法二：systemd 服务（标准方式）

这是 Linux 标准开机自启方案，适用于需要精细控制的守护进程。

### 创建服务文件

创建 `/etc/systemd/system/my-service.service`：

```ini
[Unit]
Description=My Service Description
After=network.target

[Service]
ExecStart=/path/to/your/command
Restart=on-failure
User=root
StandardOutput=append:/var/log/my-service.log
StandardError=append:/var/log/my-service.log

[Install]
WantedBy=multi-user.target
```

### 启用并启动

```bash
sudo systemctl daemon-reload
sudo systemctl enable my-service.service
sudo systemctl start my-service.service
sudo systemctl status my-service.service
```

### 常用管理命令

| 命令 | 说明 |
|------|------|
| `systemctl enable <service>` | 启用开机自启 |
| `systemctl disable <service>` | 禁用开机自启 |
| `systemctl start <service>` | 立即启动 |
| `systemctl stop <service>` | 停止服务 |
| `systemctl status <service>` | 查看状态 |
| `systemctl restart <service>` | 重启服务 |
| `journalctl -u <service>` | 查看日志 |

## 方法三：rc.local

`/etc/rc.local` 是传统的 SysV 兼容启动脚本，由 `rc-local.service` 驱动。

### 编辑 rc.local

```bash
sudo nano /etc/rc.local
```

### 格式示例

```bash
#!/bin/bash

# 开机延迟 10 秒后执行
nohup sh -c 'sleep 10 && your_command' > /dev/null 2>&1 &

exit 0
```

> **注意**：当前系统的 `/etc/rc.local` 已被配置为自动执行 `/etc/aidlux/` 下的所有脚本，建议优先使用方法一，而不是直接修改 `rc.local`。

## 方法四：桌面环境自启动（GUI 程序）

适用于图形界面程序（如桌面应用、托盘程序等）。

### 创建 .desktop 文件

用户级（推荐）：`~/.config/autostart/my-app.desktop`

```ini
[Desktop Entry]
Type=Application
Name=My App
Exec=/path/to/your/app
Terminal=false
X-GNOME-Autostart-enabled=true
```

系统级：`/etc/xdg/autostart/my-app.desktop`

```bash
cp my-app.desktop ~/.config/autostart/
```

### 当前用户已配置的自启动项

```bash
ls ~/.config/autostart/
```

## 方法五：cron @reboot

适用于定时任务风格的启动项，简单直接。

### 配置方式

```bash
crontab -e
```

添加一行：

```
@reboot /path/to/your/script.sh
```

### 查看当前 cron 任务

```bash
crontab -l
```

### 注意事项

- `@reboot` 在系统启动时执行，但环境变量可能与登录 shell 不同
- 建议在脚本中设置完整路径或 `PATH` 环境变量
- 使用 `systemctl status cron` 确认 cron 服务正常运行

## 方法六：shell 配置文件（.bashrc / .profile）

仅适用于 **用户登录时** 运行，不适合守护进程。

### 文件说明

| 文件 | 触发时机 | 适用场景 |
|------|----------|----------|
| `~/.bashrc` | 交互式 bash 启动时 | 别名、函数、环境变量 |
| `~/.profile` | 登录 Shell 启动时 | 环境变量、登录一次性任务 |
| `~/.zshrc` | zsh 启动时 | zsh 相关配置 |

### 示例（.bashrc 尾部追加）

```bash
# 用户登录时提示
echo "Welcome, $(whoami)!"

# 启动一个后台任务（仅在交互式 shell 中）
if [[ -z "$STARTED_BACKGROUND_TASK" ]]; then
    export STARTED_BACKGROUND_TASK=1
    nohup some_command &>/dev/null &
fi
```

### 注意事项

- 仅当用户登录（SSH、图形界面登录、新终端）时触发
- 不适合需要开机即运行的系统级服务
